User API

A read-only user is created with the /settings/readOnlyUser URI endpont. Only one read-only user can be created.

Description

Table 1. User endpoints
HTTP method URI path Description Parameters
POST /settings/readOnlyUser Creates the read-only user username, password, just_validate
PUT /settings/readOnlyUser Changes the read-only user password password
DELETE /settings/readOnlyUser Deletes the user none
GET /settings/readOnlyAdminName Retrieves the read-only username none
  1. Creating a read-only user
    A read-only user is create with the POST /settings/readOnlyUser HTTP method and URI.
  2. Getting the read-only user name
    The read-only user name is retrieved with the GET /settings/readOnlyAdminName HTTP method and URI.
  3. Changing the user password
    The user password is changed with the PUT /settings/readOnlyUser HTTP method and URI.
  4. Deleting the read-only user
    The read-only user is removed with the DELETE /settings/readOnlyUser HTTP method and URI.
